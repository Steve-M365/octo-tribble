## Interview Agenda

### 1. Current Process Understanding (15 minutes)

#### Current Workflow
- **Q1**: When you need to run a script or automated task, what do you do today?
  - Who do you contact?
  - How do you submit the request?
  - What information do you need to provide?
  - How long does it typically take?

- **Q2**: What types of automated tasks do you regularly need? (Examples)
  - [ ] Password resets
  - [ ] User account creation/modification
  - [ ] System health checks
  - [ ] Report generation
  - [ ] File cleanup/organization
  - [ ] Application restarts
  - [ ] Data backups
  - [ ] Other: ___________

- **Q3**: What frustrates you most about the current process?
  - Wait times
  - Complex request procedures
  - Lack of status updates
  - Difficulty getting help
  - Unclear results

### 2. User Experience Preferences (20 minutes)

#### Interface Design
- **Q4**: What devices do you typically use for work tasks?
  - [ ] Desktop computer
  - [ ] Laptop
  - [ ] Tablet
  - [ ] Mobile phone
  - [ ] Multiple devices

- **Q5**: What software interfaces do you find easy to use? (Examples: Outlook, SharePoint, etc.)
  - What makes them easy?
  - What features do you like?
  - What would you want to avoid?

#### Task Execution Preferences
- **Q6**: When running an automated task, what information would you want to see?
  - [ ] Clear description of what the task does
  - [ ] How long it will take
  - [ ] What impact it might have
  - [ ] Who to contact if something goes wrong
  - [ ] Step-by-step progress
  - [ ] Final results summary

- **Q7**: How would you prefer to provide input for tasks?
  - [ ] Simple text boxes
  - [ ] Dropdown menus
  - [ ] File upload buttons
  - [ ] Calendar/date pickers
  - [ ] Checkboxes for options
  - [ ] Guided wizards

#### Help and Support
- **Q8**: When you need help with a new system, what works best for you?
  - [ ] Built-in help text
  - [ ] Video tutorials
  - [ ] Step-by-step guides
  - [ ] Live chat support
  - [ ] Phone support
  - [ ] Email support
  - [ ] In-person training

### 3. Specific Use Cases (15 minutes)

#### Scenario-Based Questions
*Present 2-3 relevant scenarios based on their role*

**Scenario 1**: [Customize based on user's department]
"Imagine you need to [specific task]. Walk me through how you'd want this to work."
- What steps would you expect?
- What information would you need to provide?
- How would you know it was successful?
- What would concern you about this process?

**Scenario 2**: [Another relevant scenario]
- Same follow-up questions

#### Error Handling
- **Q9**: If something goes wrong with an automated task, what would you want to happen?
  - Immediate notification method
  - Information about what went wrong
  - Steps to fix or retry
  - Who to escalate to

- **Q10**: How comfortable are you with technology in general?
  - Very comfortable / Somewhat comfortable / Not very comfortable
  - What makes you nervous about new systems?
  - What gives you confidence?

### 4. Security and Approval Workflows (10 minutes)

#### Access and Permissions
- **Q11**: Some tasks might need special approval before running. How should this work?
  - Automatic approval for routine tasks
  - Manager approval for certain tasks
  - IT approval for system changes
  - Email notifications for approvals

- **Q12**: How should you be notified about task results?
  - [ ] Email summary
  - [ ] Dashboard notification
  - [ ] Text message
  - [ ] System popup
  - [ ] No notification needed

#### Security Comfort Level
- **Q13**: What makes you feel secure when using work systems?
  - Clear login process
  - Visible security indicators
  - Audit trails
  - Limited access to sensitive functions
